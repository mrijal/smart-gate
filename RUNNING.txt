RUNNING GUIDE - Smart Gate Application
=======================================

Project Structure:
- backend/  - FastAPI + InsightFace + MediaPipe (Python)
- frontend/ - Next.js (TypeScript/React)
- esp32/    - Arduino code for ESP32 hardware

---

PREREQUISITES
=============

1. Python 3.10+ (backend)
2. Node.js 18+ (frontend)
3. CUDA Toolkit 12.x + cuDNN (for GPU acceleration - optional but recommended)
4. MySQL/MariaDB database
5. MQTT Broker (shiftr.io cloud or local Mosquitto)
6. Camera (laptop webcam or USB/IP camera)

---

QUICK START (Development)
=========================

---------------------------------

### 1. Database Setup
```bash
# Install MySQL/MariaDB, create database
mysql -u root -p
CREATE DATABASE smart_gate;
CREATE USER 'gate_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON smart_gate.* TO 'gate_user'@'localhost';
FLUSH PRIVILEGES;
```

### 2. Backend Setup
```bash
cd backend

# Install dependencies
pip install -r requirements.txt

# Download MediaPipe hand landmarker model (REQUIRED for gesture detection)
# This is downloaded automatically on first run, or manually:
python -c "
import urllib.request
url = 'https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/1/hand_landmarker.task'
urllib.request.urlretrieve(url, 'hand_landmarker.task')
print('Model downloaded')
"

# Enable GPU (CRITICAL for performance - 3-5x speedup)
# PowerShell:
$env:USE_GPU="true"
$env:CUDA_VISIBLE_DEVICES="0"

# Or create .env file:
echo USE_GPU=true > .env
echo CUDA_VISIBLE_DEVICES=0 >> .env

# Configure database (edit backend/app/database/database.py or use env vars)
# DATABASE_URL=mysql+pymysql://gate_user:your_password@localhost/smart_gate

# Run backend
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 3. Frontend Setup
```bash
cd frontend

# Install dependencies
npm install

# Run development server
npm run dev
```

### 4. MQTT Broker (shiftr.io)
```bash
# shiftr.io (Cloud MQTT - no local broker needed)
# 1. Create namespace at https://shiftr.io/
# 2. Get credentials: Namespace, Token (as password), Key (as username)

# Or use local Mosquitto for development:
# docker run -d -p 1883:1883 -p 9001:9001 eclipse-mosquitto
```

ENVIRONMENT VARIABLES
=====================

Create `backend/.env`:
```env
# Database
DATABASE_URL=mysql+pymysql://gate_user:your_password@localhost/smart_gate

# GPU (enable for 3-5x speedup)
USE_GPU=true
CUDA_VISIBLE_DEVICES=0

# MQTT (shiftr.io Cloud Broker)
MQTT_BROKER=your-namespace.cloud.shiftr.io
MQTT_PORT=1883
MQTT_USERNAME=your-key
MQTT_PASSWORD=your-token
# For TLS (port 8883):
# MQTT_BROKER=your-namespace.cloud.shiftr.io
# MQTT_PORT=8883
# MQTT_TLS=true

# Camera (0 = default webcam, or RTSP URL)
CAMERA_SOURCE=0
```

Create `frontend/.env.local`:
```env
NEXT_PUBLIC_API_URL=http://localhost:8000
NEXT_PUBLIC_WS_URL=ws://localhost:8000
```

HARDWARE INTEGRATION (ESP32)
============================

1. Open `esp32/main.ino` in Arduino IDE
2. Configure WiFi credentials:
```cpp
const char* ssid = "YOUR_WIFI";
const char* password = "YOUR_PASSWORD";
```
3. Configure MQTT for shiftr.io:
```cpp
// shiftr.io (cloud)
const char* mqtt_server = "your-namespace.cloud.shiftr.io";
const int mqtt_port = 1883;  // or 8883 for TLS
const char* mqtt_user = "your-key";      // Token key
const char* mqtt_password = "your-token"; // Token secret

// Or local Mosquitto:
// const char* mqtt_server = "192.168.x.x";  // Your PC's IP
```
4. Upload to ESP32
5. Connect relay to GPIO pin (default: GPIO 5)

TROUBLESHOOTING
===============

"InsightFace model loaded successfully on CPU"
→ GPU not enabled. Set USE_GPU=true and verify CUDA installation.

"CUDA out of memory"
→ Reduce det_size in face_service.py: `det_size=(160, 160)`

"MediaPipe HandLandmarker unavailable (ExternalFile must specify at least one of 'file_content'...)"
→ Model file missing. Run the download command in Backend Setup step 2.

Camera not found
→ Check CAMERA_SOURCE (0, 1, or RTSP URL)
→ Test: `python -c "import cv2; cap=cv2.VideoCapture(0); print(cap.read())"`

MQTT connection failed
→ Check broker IP/port, firewall, username/password

Frontend can't connect to backend
→ Check NEXT_PUBLIC_API_URL matches backend URL
→ Check CORS in backend/main.py (allow_origins=["*"])

Database connection error
→ Verify DATABASE_URL format: mysql+pymysql://user:pass@host/db
→ Run migrations: tables auto-create on startup

Gesture detection triggers on face/random objects
→ MediaPipe not loaded - see "MediaPipe HandLandmarker unavailable" above
→ Ensure hand_landmarker.task is in backend/ directory

Gate opens but frontend status doesn't update
→ Check WebSocket connection (F12 Network tab)
→ Verify ESP32 publishes to "gate/status" topic
→ Check MQTT client on_message for "gate/status"

PERFORMANCE TIPS
================

For RTX 3050 6GB (see OPTIMIZATION.txt):
1. USE_GPU=true (3-5x faster)
2. det_size=(320, 320) in face_service.py
3. Camera 640x480, JPEG 50, ~30 FPS in camera.py
4. Gesture detection runs at 1 FPS when known user present
5. Face recognition runs async every 2 seconds

Expected: 25-30 FPS with GPU, 5-6 FPS CPU-only

API ENDPOINTS
=============
- GET  /api/video_feed     - MJPEG video stream
- GET  /api/users          - List registered users
- POST /api/users/register - Register new face
- GET  /api/logs           - Access logs
- POST /api/gate/open      - Manual gate open
- POST /api/gate/close     - Manual gate close
- WS   /ws                 - Real-time updates (gate_status, access_logs)

MQTT TOPICS
===========
- gate/open          - Backend → ESP32: Open gate command
- gate/close         - Backend → ESP32: Close gate command
- gate/status        - ESP32 → Backend: Gate state (OPEN/CLOSED)
- device/heartbeat   - ESP32 → Backend: Device online status
- gate/auth/request  - ESP32 → Backend: RFID scan for verification

GESTURE DETECTION
=================
The system uses MediaPipe HandLandmarker (Tasks API) for reliable gesture detection:
- Detects 5 fingers raised (open hand / high-5)
- Only activates when a known face is recognized
- Runs at 1 FPS to avoid spamming MQTT
- Cooldown: 5 seconds between gate opens

Camera positioning: Place hand in right-side ROI box shown on video feed
